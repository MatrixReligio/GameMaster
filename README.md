## What it does

GameMaster turns the command-line dance of running Windows games on a Mac into
a few clicks:

- **One-click Steam** — install the Windows version of Steam into a private
  environment and start playing your existing library.
- **Any Windows program** — drag any `.exe` or `.msi` onto the window to run it
  or add it to your library.
- **Metal-native DirectX translation** — DirectX 11 and 12 games run through
  Apple's D3DMetal (Game Porting Toolkit) or the open-source DXMT, depending
  on the runtime — Metal-native performance either way.
- **Simple by default, powerful when you need it** — sensible defaults up front;
  Retina, sync mode, MetalFX, ray tracing, and per-bottle environment variables
  are one disclosure triangle away.
- **Native macOS** — SwiftUI, follows the Human Interface Guidelines, light/dark
  aware, localized in English, Simplified & Traditional Chinese, Japanese, and
  Korean.

## Requirements

- **macOS 26 (Tahoe) or later**
- **Apple Silicon** (M1 or newer)
- **Rosetta 2** — GameMaster prompts you if it's missing
  (`softwareupdate --install-rosetta`)

## Install

1. Download the latest `GameMaster-x.y.z.dmg` from
   [Releases](https://github.com/MatrixReligio/GameMaster/releases/latest).
2. Open the DMG and drag **GameMaster** to Applications.
3. Launch it. The first-run wizard downloads the Windows runtime and walks you
   through everything.

The app updates itself automatically (Sparkle, signed & notarized by Apple).

## How the graphics layers work

GameMaster bundles no runtime — it downloads them at first run, each pinned to
an exact URL and SHA-256 digest:

- The **default runtime** is the community *Game Porting Toolkit* build by
  [Gcenx](https://github.com/Gcenx/game-porting-toolkit), fetched directly from
  that project's releases. As packaged by that project, it includes Apple's
  **D3DMetal** evaluation libraries, so DirectX 11/12 games work out of the box.
- **Steam bottles** run on an open-source runtime assembled from a
  [Sikarugir](https://github.com/Sikarugir-App) CrossOver-derived Wine engine
  and [DXMT](https://github.com/3Shain/dxmt) (Direct3D 10/11 → Metal, LGPL) —
  `scripts/assemble-steam-runtime.sh` documents the exact pinned sources.

Apple's D3DMetal libraries are proprietary; this repository and GameMaster's
own release assets never include them. To update to Apple's very latest
graphics layer, GameMaster guides you to download the free *"Evaluation
environment for Windows games"* from
[Apple Developer](https://developer.apple.com/download/all/) (a free Apple ID
is enough) and imports it locally — the import verifies the libraries are
signed by Apple, and they never leave your machine.

## Building from source

```bash
brew install xcodegen swiftlint swiftformat
git clone https://github.com/MatrixReligio/GameMaster.git
cd GameMaster
swift test          # run the core test suite
xcodegen generate   # produce GameMaster.xcodeproj
open GameMaster.xcodeproj
```

The pure-logic core lives in a Swift package (`Sources/GM*`) covered by an
extensive test suite; the SwiftUI app shell is generated by XcodeGen from
`project.yml`.

## Architecture

| Module | Responsibility |
|--------|----------------|
| `GMModel` | Domain types (bottles, programs, runtimes) — pure values |
| `GMSystem` | Process / download / disk-image / hashing primitives (protocol-injected) |
| `GMRuntime` | Wine runtime install pipeline + Apple D3DMetal import |
| `GMBottles` | Bottle lifecycle, environment composition, registry tweaks |
| `GMLaunch` | Launching programs, logging, process control |
| `GMApps` | One-click installers, program library, app-wide state |
